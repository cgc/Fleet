23  /*
24  * A grammar stores all of the rules associated with any kind of nonterminal and permits us
25  * to sample as well as compute log probabilities.
26  * A grammar also is nwo required to store them in a fixed (sorted) order that is guaranteed
27  * not to change, adn that puts terminals first and (lower priority) high probability first
28  * as determined by Rule's sort order.
29  *
30  * Note that Primitives are used to initialize a grammar, and they get "parsed" by Gramar.add
31  * to store them in the right places according to their return types (the index comes from
32  * the index of each type in GRAMMAR_TYPES).
33  * The trees that Grammar generates use nt<T>() -> size_t to represent types, not the types
34  * themselves.
35  * The trees also use Primitive.op (a size_t) to represent operations
36  *
37  * The template args GRAMMAR_TYPES stores the types used in this grammar. This is inherited
38  * by LOTHypothesis, which also passes them along to VirtualMachineState (in the form of
39  * a Tuple). So the types used and order are fixed/standardized in the grammar
40  */
